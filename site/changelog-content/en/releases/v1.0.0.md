# Release v1.0.0

**Release Date**: 2025-02-13  
**Version Type**: Initial Release (Major)

---

## 🎉 Overview

Version 1.0.0 is the inaugural release of the N-Body Particle Simulation System, providing a complete, production-ready framework for large-scale particle simulations with real-time visualization on NVIDIA GPUs.

### Key Features

| Feature | Description |
|---------|-------------|
| 🚀 GPU Acceleration | CUDA-based parallel computation supporting millions of particles |
| 🔬 Multiple Algorithms | Direct N², Barnes-Hut O(N log N), Spatial Hash O(N) |
| 🎨 Real-time Visualization | Zero-copy CUDA-OpenGL interop at 60+ FPS |
| ⚛️ Energy Conservation | Symplectic Velocity Verlet integrator |
| 🧪 Comprehensive Testing | Google Test + RapidCheck property-based testing |

## ✨ Added

### Force Calculation Algorithms

#### Direct N² (O(N²))
- Exact pairwise force calculation
- Shared memory tiling for GPU efficiency
- Configurable thread block size
- Best for: N < 10,000 particles

#### Barnes-Hut (O(N log N))
- Octree-based spatial decomposition
- Morton code sorting (Z-order curve)
- Configurable opening angle (θ)
- Best for: Large-scale gravitational simulations

#### Spatial Hash (O(N))
- Uniform grid for short-range forces
- 27-cell neighbor search
- Configurable cell size and cutoff
- Best for: Molecular dynamics, SPH fluid simulation

### Time Integration

- **Velocity Verlet** symplectic integrator
- Second-order accuracy
- Single force evaluation per step
- Long-term energy stability

### Rendering System

- OpenGL 3.3+ point sprite rendering
- CUDA-OpenGL zero-copy interop
- Real-time camera controls (orbit, zoom)
- Depth-based particle coloring

### Particle Distributions

- **Uniform**: Random within bounding box
- **Spherical**: Uniform within sphere
- **Disk**: Flat disk with initial rotation (galaxy simulation)

### Project Infrastructure

- CMake 3.18+ build system
- Google Test integration
- RapidCheck property-based testing
- MIT License
- `.gitignore` with CUDA/OpenGL rules
- `.editorconfig` for consistent formatting

## 📝 API Highlights

```cpp
#include "nbody/particle_system.hpp"

using namespace nbody;

// Configure and initialize
SimulationConfig config;
config.particle_count = 100000;
config.force_method = ForceMethod::BARNES_HUT;

ParticleSystem system;
system.initialize(config);

// Run simulation
for (int i = 0; i < 1000; ++i) {
    system.update(0.001f);
}

// Save state
system.saveState("checkpoint.nbody");
```

## ⚠️ Known Limitations

1. **CUDA Required**: Cannot build without NVIDIA GPU and CUDA toolkit
2. **Single GPU**: Multi-GPU support not yet implemented
3. **Long-range forces only**: Spatial Hash assumes short-range interactions
4. **No collisions**: Particles can pass through each other

## 📦 Installation

### Prerequisites

- CUDA Toolkit 11.0+
- CMake 3.18+
- OpenGL 3.3+
- GLFW, GLEW, GLM
- C++17 compiler

### Build

```bash
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
cmake --build . -j$(nproc)
```
