## MODIFIED Requirements

### Requirement: Build Simplicity
The system SHALL provide a simple, canonical build path that also supports validation automation, headless/core verification, and editor/LSP integration.

#### Scenario: Canonical build path
- **WHEN** a contributor follows the primary build instructions
- **THEN** the repository SHALL provide one clearly documented CMake-based build path that remains consistent across docs, scripts, and automation

#### Scenario: Dependency versions are controlled
- **WHEN** the build depends on fetched tools, libraries, or actions
- **THEN** their versions SHALL be pinned or explicitly bounded in a way that is documented and reproducible

#### Scenario: LSP derives from the primary build
- **WHEN** editor tooling is configured for C++ or CUDA development
- **THEN** the recommended setup SHALL use artifacts generated by the canonical build path, including `compile_commands.json` or an equivalent compile database

#### Scenario: Headless validation path is supported
- **WHEN** contributors or CI need to validate core simulation behavior without a graphics stack
- **THEN** the build system SHALL provide a supported core/headless path derived from the canonical CMake configuration

## ADDED Requirements

### Requirement: Performance Regression Visibility
The system SHALL make major performance regressions observable through repeatable measurements.

#### Scenario: Repeatable performance measurement
- **WHEN** a contributor runs the supported benchmark workflow
- **THEN** the repository SHALL provide repeatable measurement surfaces for major simulation subsystems

#### Scenario: Performance context is preserved
- **WHEN** benchmark results are recorded or reviewed
- **THEN** the results SHALL include enough configuration context to compare runs meaningfully
