# 能力：力计算

## 概述

提供高效的力计算算法，包括用于 O(N) 短程作用力的空间哈希以及 Velocity Verlet 数值积分。

---

## 新增需求

### 需求：空间哈希网格

系统应将三维空间划分为均匀网格单元，以实现高效的邻居搜索。

#### 场景：单元分配
- **当** 粒子分布在空间中时
- **则** 每个粒子应被分配到且仅分配到一个网格单元

#### 场景：单元边界
- **当** 粒子被分配到某个单元时
- **则** 该单元边界应包含该粒子的位置

---

### 需求：邻居搜索

系统应仅处理相邻单元中的粒子以计算短程作用力。

#### 场景：截断半径
- **当** 计算短程作用力时
- **则** 仅应考虑位于截断半径内的粒子

#### 场景：邻近单元
- **当** 执行邻居搜索时
- **则** 仅应检查 3×3×3 的单元邻域

#### 场景：可配置参数
- **当** 使用空间哈希方法时
- **则** 系统应支持可配置的单元大小和截断半径

---

### 需求：Velocity Verlet 积分

系统应实现 Velocity Verlet 辛积分器。

#### 场景：位置更新
- **当** 计算一个时间步时
- **则** 位置应按 x(t+dt) = x(t) + v(t)·dt + ½·a(t)·dt² 更新

#### 场景：速度更新
- **当** 计算出新的加速度时
- **则** 速度应按 v(t+dt) = v(t) + ½·(a_old + a_new)·dt 更新

#### 场景：单次力评估
- **当** 执行积分步骤时
- **则** 每一步仅应需要一次力评估

---

### 需求：时间步配置

系统应支持可配置的时间步。

#### 场景：自定义时间步
- **当** 用户指定时间步值时
- **则** 积分器应使用该值进行仿真

#### 场景：并行更新
- **当** 执行积分时
- **则** 所有粒子应在 GPU 上并行更新

---

### 需求：能量守恒

系统应提供能量守恒指标。

#### 场景：能量漂移测量
- **当** 仿真运行 1000 步时
- **则** 能量漂移应小于 1%

#### 场景：相空间保持
- **当** 使用辛积分器时
- **则** 相空间体积应被保持
