# 能力：模拟控制

## 概述

提供模拟参数的运行时控制，包括暂停/恢复、状态序列化和全面的错误处理。

---

## 新增需求

### 需求：模拟状态控制

系统应支持暂停、恢复和重置操作。

#### 场景：暂停模拟
- **当** 用户请求暂停时
- **则** 模拟应停止推进同时保持当前状态

#### 场景：恢复模拟
- **当** 用户在暂停后请求恢复时
- **则** 模拟应从暂停状态继续

#### 场景：重置模拟
- **当** 用户请求重置时
- **则** 模拟应返回初始条件

#### 场景：状态保持
- **当** 模拟暂停时
- **则** 状态应保持不变直到恢复或重置

---

### 需求：运行时力方法切换

系统应允许运行时切换力计算方法。

#### 场景：方法更改
- **当** 用户在运行时选择不同的力方法时
- **则** 系统应无缝切换到新方法

#### 场景：可用方法
- **当** 用户查询可用方法时
- **则** 应列出直接 N²、Barnes-Hut 和空间哈希

---

### 需求：状态序列化

系统应支持保存和加载模拟状态。

#### 场景：保存状态
- **当** 用户请求保存状态时
- **则** 所有粒子位置、速度和模拟参数应写入文件

#### 场景：加载状态
- **当** 用户请求加载状态时
- **则** 模拟应从保存的文件恢复

#### 场景：往返完整性
- **当** 状态保存后再加载时
- **则** 加载的状态应完全等于原始状态

---

### 需求：运行时参数调整

系统应允许运行时调整参数。

#### 场景：时间步调整
- **当** 用户在模拟期间更改时间步时
- **则** 新时间步应立即生效

#### 场景：Theta 调整
- **当** 用户更改 Barnes-Hut theta 参数时
- **则** 近似精度应相应更新

---

### 需求：CUDA 错误报告

系统应报告详细的 CUDA 初始化错误。

#### 场景：未找到设备
- **当** 未找到支持 CUDA 的设备时
- **则** 应显示有意义的错误消息

#### 场景：初始化失败
- **当** CUDA 初始化失败时
- **则** 应记录详细的错误信息

---

### 需求：输入验证

系统应验证所有用户输入。

#### 场景：无效粒子数量
- **当** 用户指定负数或零粒子数量时
- **则** 系统应拒绝输入并显示错误消息

#### 场景：无效时间步
- **当** 用户指定负时间步时
- **则** 系统应拒绝输入并显示错误消息

#### 场景：拒绝时保持状态
- **当** 无效输入被拒绝时
- **则** 系统状态应保持不变

---

### 需求：GPU 内存验证

系统应在分配前检查 GPU 内存可用性。

#### 场景：内存检查
- **当** 请求大量分配时
- **则** 系统应验证有足够的 GPU 内存可用

#### 场景：内存耗尽
- **当** GPU 内存不足时
- **则** 应显示有意义的错误消息

---

### 需求：有意义的错误消息

系统应提供有意义的错误消息。

#### 场景：错误上下文
- **当** 发生错误时
- **则** 错误消息应包含关于失败内容和原因的上下文

#### 场景：恢复建议
- **当** 可能时
- **则** 错误消息应建议恢复操作
