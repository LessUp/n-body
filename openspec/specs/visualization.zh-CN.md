# 能力：可视化

## 概述

使用 CUDA-OpenGL 互操作提供粒子模拟的实时可视化，实现零拷贝渲染。

---

## 新增需求

### 需求：CUDA-OpenGL 互操作

系统应在 CUDA 中注册 OpenGL 缓冲区以实现零拷贝数据共享。

#### 场景：缓冲区注册
- **当** 可视化初始化时
- **则** OpenGL 顶点缓冲区应在 CUDA 中注册

#### 场景：直接映射
- **当** 粒子位置更新时
- **则** 位置应直接映射到顶点缓冲区，无需 CPU 传输

#### 场景：零拷贝渲染
- **当** 渲染帧时
- **则** 渲染循环中不应发生 CPU-GPU 数据传输

#### 场景：动态缓冲区调整
- **当** 粒子数量变化时
- **则** 系统应支持动态缓冲区调整

---

### 需求：点精灵渲染

系统应将粒子显示为点精灵。

#### 场景：粒子可视化
- **当** 模拟运行时
- **则** 每个粒子应渲染为点精灵

#### 场景：点大小
- **当** 渲染粒子时
- **则** 点大小应可配置

---

### 需求：相机控制

系统应支持相机轨道和缩放控制。

#### 场景：轨道控制
- **当** 用户拖动鼠标时
- **则** 相机应围绕模拟中心旋转

#### 场景：缩放控制
- **当** 用户滚动鼠标滚轮时
- **则** 相机应放大或缩小

#### 场景：视图矩阵
- **当** 相机移动时
- **则** 视图矩阵应正确变换坐标

---

### 需求：颜色映射

系统应支持多种颜色模式（深度、速度、密度）。

#### 场景：深度颜色模式
- **当** 选择深度颜色模式时
- **则** 粒子颜色应反映与相机的距离

#### 场景：速度颜色模式
- **当** 选择速度颜色模式时
- **则** 粒子颜色应反映速度大小

#### 场景：密度颜色模式
- **当** 选择密度颜色模式时
- **则** 粒子颜色应反映局部粒子密度

#### 场景：有效颜色范围
- **当** 任何颜色模式激活时
- **则** 颜色应在有效 RGB 范围 [0, 1] 内

---

### 需求：模拟统计显示

系统应显示模拟统计信息。

#### 场景：FPS 显示
- **当** 模拟运行时
- **则** 应显示每秒帧数

#### 场景：粒子数量显示
- **当** 模拟运行时
- **则** 应显示当前粒子数量

---

### 需求：实时性能

系统应为 100 万粒子保持 30+ FPS。

#### 场景：帧率目标
- **当** 使用 Barnes-Hut 模拟 100 万粒子时
- **则** 帧率应至少为 25 FPS

#### 场景：高性能模式
- **当** 使用空间哈希模拟 100 万粒子时
- **则** 帧率应至少为 60 FPS
