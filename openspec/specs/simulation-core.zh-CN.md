# 能力：模拟核心

## 概述

管理 N 体系统状态、粒子数据和力计算算法，包括用于 O(N log N) 性能的 Barnes-Hut 算法。

---

## 新增需求

### 需求：粒子数据布局

系统应使用结构数组（SoA）布局以实现 GPU 内存合并访问。

#### 场景：内存合并访问
- **当** 粒子数据在 GPU 上被访问时
- **则** 系统应将数据按属性排列为连续数组（pos_x、pos_y、pos_z、vel_x、vel_y、vel_z 等）

#### 场景：大规模支持
- **当** 模拟初始化时
- **则** 系统应支持至少 1000 万个粒子

---

### 需求：粒子属性

系统应存储每个粒子的位置、速度、加速度和质量。

#### 场景：完整状态存储
- **当** 粒子被创建时
- **则** 系统应为位置（3N）、速度（3N）、加速度（6N - 当前 + 旧值）和质量（N）分配内存

#### 场景：内存效率
- **当** 模拟 100 万个粒子时
- **则** 总内存使用应小于 200 MB

---

### 需求：初始分布

系统应提供可配置的初始分布（均匀、球形、圆盘）。

#### 场景：均匀分布
- **当** 用户指定均匀分布时
- **则** 粒子应在指定边界内均匀分布

#### 场景：球形分布
- **当** 用户指定球形分布时
- **则** 粒子应在具有可配置半径的球体内分布

#### 场景：圆盘分布
- **当** 用户指定圆盘分布时
- **则** 粒子应以旋转圆盘模式分布

---

### 需求：引力计算

系统应使用牛顿定律计算引力。

#### 场景：成对力
- **当** 计算两个粒子之间的力时
- **则** 力的大小应近似 G·m₁·m₂/(r² + ε²)，其中 ε 为软化参数

#### 场景：软化参数
- **当** 粒子非常接近时
- **则** 系统应应用软化以防止数值奇点

---

### 需求：GPU 并行计算

系统应使用 GPU 并行计算，每个粒子一个线程。

#### 场景：线程分配
- **当** 执行力计算时
- **则** 每个粒子应由专用的 GPU 线程处理

#### 场景：共享内存缓存
- **当** 计算力时
- **则** 系统应使用 CUDA 共享内存进行数据缓存

#### 场景：硬件优化
- **当** 计算平方根倒数时
- **则** 系统应使用硬件 rsqrt 以提高性能

---

### 需求：Barnes-Hut 算法

系统应实现 Barnes-Hut 算法以实现 O(N log N) 力计算。

#### 场景：八叉树构建
- **当** 选择 Barnes-Hut 方法时
- **则** 系统应从粒子位置构建八叉树

#### 场景：质心计算
- **当** 构建树节点时
- **则** 每个节点应存储所含粒子的质心和总质量

#### 场景：Theta 参数
- **当** 遍历树进行力计算时
- **则** 系统应使用可配置的 θ 参数控制近似精度

#### 场景：树重建
- **当** 模拟推进到下一步时
- **则** 系统应重建八叉树

#### 场景：算法切换
- **当** 用户请求不同的力方法时
- **则** 系统应支持运行时在直接 N²、Barnes-Hut 和空间哈希之间切换
